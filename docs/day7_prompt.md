### Phase B — Resend + React Email setup (~30 min)

1. `npm install resend @react-email/components @react-email/render`.
2. Create `src/lib/email/client.ts`:
   ```ts
   import { Resend } from 'resend';
   const apiKey = process.env.RESEND_API_KEY;
   if (!apiKey) console.warn('[email] RESEND_API_KEY missing — sends will fail.');
   export const resend = new Resend(apiKey);
   export const FROM = process.env.RESEND_FROM_EMAIL ?? 'MVS <onboarding@resend.dev>';
   ```
3. Create `src/lib/email/send.ts` with a typed sender:
   ```ts
   import { render } from '@react-email/render';
   import { resend, FROM } from './client';

   export interface SendArgs {
     to: string;
     subject: string;
     react: React.ReactElement;
     replyTo?: string;
     idempotencyKey?: string;   // Resend supports headers for de-dupe
   }
   export async function sendEmail(args: SendArgs) {
     const html = await render(args.react);
     const text = await render(args.react, { plainText: true });
     const result = await resend.emails.send({
       from: FROM,
       to: args.to,
       subject: args.subject,
       html, text,
       replyTo: args.replyTo,
       headers: args.idempotencyKey ? { 'Idempotency-Key': args.idempotencyKey } : undefined,
     });
     if (result.error) {
       console.error('[email] send failed', { to: args.to, subject: args.subject, error: result.error });
       throw new Error(`Resend send failed: ${result.error.message}`);
     }
     return result.data!.id;
   }
   ```

#### D.2 Cron endpoint (~30 min)

1. Create `src/app/api/cron/send-reminders/route.ts`:
   - GET handler.
   - Auth: check `request.headers.get('authorization') === \`Bearer \${process.env.CRON_SECRET}\`` — return 401 otherwise. Vercel Cron sends this header automatically when configured (see vercel.json below).
   - Query enrollments needing emails:
     - **Missing-invite catch-up:** `invited_email_sent_at IS NULL AND completed_at IS NULL` → fire the AssignmentInvite/PostInvite (whichever fits phase), then stamp `invited_email_sent_at`.
     - **Pre-reminder:** `invited_email_sent_at IS NOT NULL AND reminder_sent_at IS NULL AND completed_at IS NULL AND due_at IS NOT NULL AND due_at - now() <= interval '3 days' AND due_at > now() AND phase IN ('pre','practice')` → fire PreReminder, stamp `reminder_sent_at`.
     - **Post-reminder:** same predicate but `phase = 'post'` → PostReminder.
   - Process in batches of N (start with 50). Cap total per run at 200.
   - Return JSON summary: `{ scanned, sent: { invites, pre_reminders, post_reminders }, errors }`.

2. Create `vercel.json` at repo root (or update if exists):
   ```json
   {
     "crons": [
       { "path": "/api/cron/send-reminders", "schedule": "0 14 * * *" }
     ]
   }
   ```
   Schedule: 14:00 UTC daily ≈ 9 AM ET. Adjust if doctor wants a different send window.

3. **Vercel plan note:** Vercel Hobby plan limits cron to once-daily and only allows 2 active crons. We need 1, so Hobby is fine. If the user is on a non-Pro plan and we need more granularity, log to `needs_human.md` — alternative is Supabase Edge Function with `pg_cron`.
